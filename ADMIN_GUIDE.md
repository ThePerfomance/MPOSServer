# 📘 Руководство по использованию админ-панели и ролевой модели

## 🎯 Роли пользователей

В системе реализованы три роли:

| Роль | Описание | Права доступа |
|------|----------|---------------|
| **student** | Студент | Просмотр контента, прохождение тестов |
| **teacher** | Преподаватель | Создание предметов, блоков, уроков, тестов, вопросов |
| **admin** | Администратор | Полный доступ + управление пользователями, типами видео, видео |

---

## 🔐 Ограничения доступа в админ-панели

### Студент (`role=student`)
- ❌ Нет доступа к админ-панели

### Преподаватель (`role=teacher`)
- ✅ **Доступно:**
  - Предметы (Subject) — просмотр, создание, редактирование
  - Тематические блоки (Block) — просмотр, создание, редактирование
  - Уроки (Lesson) — просмотр, создание, редактирование
  - Тесты (Test) — просмотр, создание, редактирование
  - Вопросы (Question) — просмотр, создание, редактирование
  - Ответы (Answer) — редактирование внутри вопросов
  - Результаты тестов (TestResult) — просмотр
  - Ответы пользователей (UserAnswer) — просмотр
  - Сессии тренажёра (TrainingSession) — просмотр
  
- ❌ **Недоступно:**
  - Пользователи (User) — только админ
  - Типы видео (VideoType) — только админ
  - Видео (Video) — только админ
  - Удаление сущностей — только админ

### Администратор (`role=admin`)
- ✅ **Полный доступ ко всем разделам**
- Управление пользователями
- Управление типами видео и видеоматериалами
- Удаление любых сущностей

## 📝 Создание предметов, тестов и тем через админ-панель

### 1. Создание предмета (Subject)
1. Войдите в админ-панель как преподаватель или админ
2. Перейдите в раздел **"Предметы"** → **"Добавить предмет"**
3. Заполните:
   - **Название**: например, "Математика"
4. Сохраните

### 2. Создание тематического блока (Block)
1. Перейдите в раздел **"Тематические блоки"** → **"Добавить блок"**
2. Заполните:
   - **Название**: например, "Алгебра"
   - **Предмет**: выберите созданный предмет
   - **Описание**: краткое описание блока
   - **Позиция**: порядок отображения (0, 1, 2...)
   - **Опубликовано**: галочка для видимости студентам
3. Сохраните

### 3. Создание урока (Lesson)
1. Перейдите в раздел **"Уроки"** → **"Добавить урок"**
2. Заполните:
   - **Название**: например, "Квадратные уравнения"
   - **Блок**: выберите созданный блок
   - **Краткое содержание**: описание урока (до 1000 символов)
   - **Видео**: выберите из загруженных (только админ может загрузить)
   - **Тест**: создайте заранее или выберите существующий
   - **Позиция**: порядок в блоке
   - **Длительность**: в секундах
   - **Опубликовано**: галочка для видимости
3. Сохраните

### 4. Создание теста (Test)
1. Перейдите в раздел **"Тесты"** → **"Добавить тест"**
2. Заполните:
   - **Название**: например, "Проверка знаний по алгебре"
   - **Описание**: опционально
   - **Длительность**: таймер в секундах (0 = без ограничений)
   - **Опубликовано**: галочка для доступности
3. Сохраните

### 5. Добавление вопросов в тест
1. Перейдите в раздел **"Вопросы"** → **"Добавить вопрос"**
2. Заполните:
   - **Текст вопроса**: формулировка
   - **Тест**: выберите созданный тест
   - **Ссылка на материал для повторения**: URL для рекомендации при ошибке
   - **Ссылка на видео для повторения**: URL видео для работы над ошибками
3. В разделе **"Варианты ответов"** добавьте 2-4 варианта:
   - **Текст ответа**: вариант ответа
   - **Правильный**: галочка для верного ответа
4. Сохраните

## ⚙️ Автоматический ML-анализ после прохождения теста

### Как это работает:

1. Студент проходит тест в мобильном приложении
2. Приложение отправляет POST-запрос на `/api/test-results/` с ответами
3. Сервер сохраняет результат и ответы
4. **Автоматически запускается ML-анализ:**
   - `analyze_weak_topics()` — определяет слабые темы
   - `generate_personalized_recommendations()` — генерирует рекомендации

### Пример запроса от мобильного приложения:

```json
POST /api/test-results/
{
  "user": "uuid-пользователя",
  "test": 1,
  "score": 75,
  "started_at": "2026-04-24T10:00:00Z",
  "answers": [
    {
      "question_id": 1,
      "chosen_answer_id": 3
    },
    {
      "question_id": 2,
      "chosen_answer_id": 1
    }
  ]
}
```

### Что происходит на сервере:

```python
# 1. Сохранение TestResult
test_result = serializer.save()

# 2. Сохранение UserAnswer (с авто-определением правильности)
for ans in answers_data:
    UserAnswer.objects.create(
        test_result=test_result,
        question_id=ans["question_id"],
        chosen_answer_id=ans["chosen_answer_id"],
        is_correct=auto_check(...)  # Если не передано
    )

# 3. ML-анализ
weak_topics = analyze_weak_topics(user_id)
recommendations = generate_personalized_recommendations(user_id)
```

## 🧪 Тестирование системы

### 1. Создание тестовых данных

```bash
# Создайте суперпользователя-админа
python manage.py createsuperuser

# Email: admin@example.com
# Пароль: admin123
# Роль: admin (выберите при создании или измените в админке)
```

### 2. Вход в админ-панель

1. Откройте `http://localhost:8000/admin/`
2. Войдите под админом
3. Создайте преподавателя:
   - Раздел "Пользователи" → "Добавить"
   - Заполните данные, выберите роль "Преподаватель"

### 3. Проверка прав доступа

- Войдите под преподавателем → убедитесь, что нет доступа к "Типы видео" и "Видео"
- Попробуйте создать предмет, блок, урок, тест → должно работать
- Попробуйте удалить → должно быть запрещено

## 🚀 Быстрый старт для демонстрации диплома

### Сценарий демонстрации:

1. **Админ создаёт структуру:**
   - Предмет: "Информатика"
   - Блок: "Основы Python"
   - Урок: "Переменные и типы данных"
   - Тест: 5 вопросов с рекомендациями

2. **Преподаватель проверяет:**
   - Заходит в админку
   - Видит предметы, блоки, уроки, тесты
   - Не видит "Видео" и "Типы видео"
   - Может добавить новый вопрос

3. **Студент проходит тест:**
   - Мобильное приложение отправляет ответы
   - Сервер автоматически анализирует ошибки
   - Генерируются персонализированные рекомендации

4. **Преподаватель видит аналитику:**
   - Результаты тестов студентов
   - Слабые темы группы
   - Рекомендации для каждого студента
