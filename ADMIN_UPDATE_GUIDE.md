## 🚀 Инструкция по запуску

### Шаг 1: Запуск базы данных и сервера

```bash
# Запуск через Docker Compose
cd /workspace
docker compose up -d db
docker compose up --build
```

Или вручную:

```bash
# Убедитесь, что PostgreSQL запущен
# Затем:
cd /workspace
python manage.py migrate
python manage.py runserver
```

### Шаг 2: Создание суперпользователя (admin)

```bash
python manage.py createsuperuser
# Email: admin@example.com
# Пароль: [ваш пароль]
# Роль: admin (выберите при создании или измените в админке)
```

### Шаг 3: Создание преподавателя через админку

1. Войдите как admin: `http://localhost:8000/admin/`
2. Раздел **"Пользователи"** → **"Добавить пользователя"**
3. Заполните:
   - Email: `teacher@example.com`
   - Имя/Фамилия
   - **Роль: Преподаватель**
   - is_staff: ✓ (для доступа в админку)
   - is_active: ✓

### Шаг 4: Проверка прав преподавателя

1. Выйдите из админки
2. Войдите как преподаватель: `teacher@example.com`
3. **Должно быть доступно:**
   - ✅ Предметы (создание, редактирование)
   - ✅ Тематические блоки
   - ✅ Уроки
   - ✅ Тесты
   - ✅ Вопросы
   - ✅ Типы видео (просмотр)
   - ✅ Видеоматериалы (создание, редактирование)
   - ✅ Группы (только просмотр)
   - ✅ Результаты тестов (просмотр)

4. **Не должно быть доступа:**
   - ❌ Удаление любых сущностей
   - ❌ Редактирование пользователей с ролью admin
   - ❌ Стандартные группы Django (auth.Group)

## 🐛 Возможные проблемы и решения

### Проблема: Не видно преподавателей в админке
**Решение:** Убедитесь, что у пользователя role='teacher' и is_staff=True

### Проблема: Видны стандартные группы Django
**Решение:** Проверьте, что применяется переопределение `get_model_perms`

### Проблема: Дашборд пустой
**Решение:** 
1. Проверьте, что пользователь имеет права teacher/admin
2. Создайте тестовые данные (предметы, тесты)

### Проблема: Не работает ML-анализ после теста
**Решение:**
1. Проверьте логи сервера
2. Убедитесь, что функция `analyze_weak_topics` вызывается в `test_results_list`

## 📝 Тестовые данные для быстрой проверки

```python
# Создать через python manage.py shell
from api.models import User, Subject, Block, Lesson, Test, Question, Answer

# Преподаватель
teacher = User.objects.create_user(
    email='teacher@demo.com',
    password='demo123',
    firstname='Иван',
    lastname='Петров',
    role='teacher',
    is_staff=True
)

# Предмет
subject = Subject.objects.create(name="Программирование на Python")

# Блок
block = Block.objects.create(
    title="Основы синтаксиса",
    subject=subject,
    position=1
)

# Урок
lesson = Lesson.objects.create(
    title="Переменные и типы данных",
    block=block,
    position=1,
    duration=300,
    is_published=True
)

# Тест
test = Test.objects.create(
    title="Проверка знаний по теме",
    duration=600,
    is_published=True
)
lesson.test = test
lesson.save()

# Вопрос
question = Question.objects.create(
    text="Какой тип данных используется для целых чисел?",
    test=test
)

# Варианты ответов
Answer.objects.create(text="int", question=question, is_correct=True)
Answer.objects.create(text="float", question=question, is_correct=False)
Answer.objects.create(text="str", question=question, is_correct=False)
Answer.objects.create(text="bool", question=question, is_correct=False)

print("✅ Тестовые данные созданы!")
```
